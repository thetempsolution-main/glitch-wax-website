Project Overview:
This project involves designing and building a basic business website for a company called Glitch Wax, which produces and sells board wax for various board sports such as skateboarding, snowboarding, and longboarding. The primary goal of the website is to provide a central hub that the client can link from their social media, allowing customers to learn about the product and make purchases. The website now functions as a simplified storefront with two products using a single combined store and order page, eliminating unnecessary steps in the buying process. External payment processing will be handled through Square rather than using a custom-built checkout system. The site will also serve as a branding tool to help establish legitimacy and increase conversions. This is a low-cost initial build intended to demonstrate value and open the door for future paid work. The project also serves as a portfolio piece and potential college-related work.

Completed Work:
A client agreement has been established where a basic website is built for free upfront, with a $50 payment if the client accepts the site, along with a right of first refusal agreement for future work. The project scope has been clearly defined to include a home page, a contact page with a form, a store page, and optional product pages. The technical approach has been decided as a static website using HTML, CSS, and JavaScript, with no custom backend or cart system, and external payment processing handled through Square checkout links or a future serverless integration.

The development environment has been set up by creating a project folder named glitch-wax-website and opening it in Visual Studio Code. The base file structure has been created, including index.html, contact.html, store.html, product-single.html, product-double.html, styles.css, and script.js. A basic homepage scaffold has been created with HTML boilerplate, and both the CSS and JavaScript files have been properly linked and tested. Live preview functionality has been set up using the Live Server extension in Visual Studio Code.

A GitHub repository has been created and configured with GitHub Pages to provide a live, shareable version of the website. This allows the client to view updates in real time via a direct link without requiring local hosting or in-person demonstrations. The deployment workflow is now established, enabling quick iteration and feedback by updating files and pushing changes to the repository.

The homepage has now been fully built out as a complete first version. It includes a hero section, a “What is board wax?” section, a redesigned feature section with supporting images, a “Why Glitch Wax?” brand section, a featured products section, and a final call-to-action section. A consistent navigation bar and footer have been implemented. A structured image system has been created using an images folder, and stock images have been added and wired into all relevant sections of the homepage. The feature section has been upgraded from simple text cards to a mixed image and content layout for a more professional appearance. The CSS has been refactored to use variables for colors, spacing, and styling, making future visual changes easy and centralized. The homepage is now visually complete and functional as a temporary but presentable solution.

The store page has been fully designed and implemented as a combined store and order page. This page displays both products side by side in a clean, card-based layout with real product images, simplified descriptions, pricing, and quantity selectors using dropdown menus. The page includes a live-updating subtotal calculation using JavaScript and a centralized order summary section. The layout has been refined to ensure consistent spacing, alignment, and visual balance between product cards and the order summary. The design prioritizes clarity, speed, and conversion by minimizing steps between product selection and checkout. The original concept of a separate order page has been removed, and the store page now serves as the single entry point for purchases.

Notes and Constraints:
The website should remain simple and quick to build, avoiding unnecessary complexity such as a full custom ecommerce system. The payment flow should rely on a streamlined store page combined with Square-hosted checkout rather than implementing a full cart and checkout system. The focus should be on clarity, usability, and conversion rather than overdesign. The current structure intentionally minimizes friction by reducing the number of steps required to make a purchase. This is intended to be a starter website rather than a final long-term solution, and the structure should remain clean, modular, and easy to update in the future. All styling decisions should remain flexible and centralized to allow for easy updates once branding is finalized.
