Project Overview:
This project involves designing and building a basic business website for a company called Glitch Wax, which produces and sells board wax for various board sports such as skateboarding, snowboarding, and longboarding. The primary goal of the website is to provide a central hub that the client can link from their social media, allowing customers to learn about the product and make purchases. The website will function as a simple storefront with two products and will integrate external payment processing through Square rather than using a custom-built checkout system. The site will also serve as a branding tool to help establish legitimacy and increase conversions. This is a low-cost initial build intended to demonstrate value and open the door for future paid work. The project also serves as a portfolio piece and potential college-related work.

Completed Work:
A client agreement has been established where a basic website is built for free upfront, with a $50 payment if the client accepts the site, along with a right of first refusal agreement for future work. The project scope has been clearly defined to include a home page, a contact page with a form, a store page, and two product pages for the single stick and double pack products. The technical approach has been decided as a static website using HTML, CSS, and JavaScript, with no custom backend or cart system, and external payment processing handled through Square checkout links. The development environment has been set up by creating a project folder named glitch-wax-website and opening it in Visual Studio Code. The base file structure has been created, including index.html, contact.html, store.html, product-single.html, product-double.html, styles.css, and script.js. A basic homepage scaffold has been created with HTML boilerplate, and both the CSS and JavaScript files have been properly linked and tested. Live preview functionality has been set up using the Live Server extension in Visual Studio Code.

A GitHub repository has been created and configured with GitHub Pages to provide a live, shareable version of the website. This allows the client to view updates in real time via a direct link without requiring local hosting or in-person demonstrations. The deployment workflow is now established, enabling quick iteration and feedback by updating files and pushing changes to the repository.

The homepage has now been fully built out as a complete first version. It includes a hero section, a “What is board wax?” section, a redesigned feature section with supporting images, a “Why Glitch Wax?” brand section, a featured products section, and a final call-to-action section. A consistent navigation bar and footer have been implemented. A structured image system has been created using an images folder, and stock images have been added and wired into all relevant sections of the homepage. The feature section has been upgraded from simple text cards to a mixed image and content layout for a more professional appearance. The CSS has been refactored to use variables for colors, spacing, and styling, making future visual changes easy and centralized. The homepage is now visually complete and functional as a temporary but presentable solution.

Remaining Tasks:
Client assets and content still need to be collected, including an official logo, finalized brand colors or style direction, finalized product pricing, product descriptions written in plain English, a refined explanation of what board wax is and why it is useful, contact email information, social media links, and Square checkout configuration details. The client has indicated that photos and videos of the actual product will be provided soon, which will replace current placeholder and stock imagery.

The remaining pages of the website still need to be built out and finalized. The store page needs to display both products in a clean layout with images, names, prices, and links to their respective product pages. Each product page needs to include images, product details, descriptions, and a buy button that routes users to a shared order page rather than directly to checkout. The contact page needs to include a working contact form along with basic contact information and social links.

A shared order page needs to be designed and implemented as a central pre-checkout screen. This page should allow users to select quantities for both products, display a subtotal for items only, and include a note indicating that tax and shipping will be calculated on the next screen. All Buy Now buttons across the site should route to this page, with optional preselection of products based on where the user came from.

A payment integration flow needs to be implemented using Square. This will require creating a secure backend or serverless function that receives product quantities from the order page, validates the input, constructs the order using trusted pricing data, and generates a Square-hosted checkout link. The user will then be redirected to Square’s hosted checkout page to complete payment.

Production hosting architecture needs to be defined. GitHub Pages will continue to be used for development preview, but the production site will be deployed on a low-cost hosting platform (likely Cloudflare) that supports the required frontend and serverless backend components for payment processing.

A full payment flow testing process needs to be implemented. This includes testing the entire flow using Square’s Sandbox environment, verifying correct order creation, redirects, and behavior under different input conditions, followed by a single real transaction in production to confirm the live system works as expected.

The navigation bar and footer should be reviewed across all pages to ensure consistency as additional pages are built. Image placeholders should be replaced with final assets once provided by the client. Existing homepage copy should also be updated to reflect client feedback emphasizing “flow,” “glide,” and a smoother, slicker feel rather than “grip” or “traction.”

Optional and Future Improvements:
Additional improvements may include mobile responsiveness refinement and testing across devices, basic search engine optimization such as titles and meta descriptions, analytics integration, domain and hosting setup, performance optimization such as image compression, and a potential maintenance or update plan if ongoing work is agreed upon. Additional UI/UX enhancements and branding refinements can be applied once the client provides more direction.

Notes and Constraints:
The website should remain simple and quick to build, avoiding unnecessary complexity such as a full custom ecommerce system. The payment flow should rely on a custom order page combined with Square-hosted checkout rather than implementing a full cart and checkout system. The focus should be on clarity, usability, and conversion rather than overdesign. This is intended to be a starter website rather than a final long-term solution, and the structure should remain clean, modular, and easy to update in the future. All styling decisions should remain flexible and centralized to allow for easy updates once branding is finalized.
